Deploying services

A deploy is the unit of change for a service. Each deploy is immutable, addressable by an id, and can be rolled back at any time. Deploys go through build → probe → release → observe, and the platform promotes traffic gradually unless you opt out.

Deploy from git
# Authenticate once
$ nc login

# Deploy current branch
$ nc deploy --service=api-gateway --env=production

Once a deploy starts, its id is stable — you can stream logs, pause the rollout, or roll back using the same reference.

Gradual rollout is default. New traffic starts at 1% and ramps over 10 minutes. Override with --rollout=instant.
Phases
  1. Build — resolve dependencies, run tests, produce image.
  2. Probe — start one instance, wait for health checks.
  3. Release — gradually shift traffic.
  4. Observe — stream metrics; auto-rollback if error budget is spent.
