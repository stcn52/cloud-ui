# Theming

Three axes of configuration: **theme**, **size** (density), and **locale**. All three flow through a single provider.

## ConfigProvider

```tsx
import { ConfigProvider, zhCN } from '@stcn52/cloud-ui'

<ConfigProvider theme="dark" size="compact" locale={zhCN}>
  <YourApp />
</ConfigProvider>
```

## Dark mode

Single attribute flip — no double stylesheet, no component-level overrides.

```tsx
<ConfigProvider theme={isDark ? 'dark' : 'light'}>
  <App />
</ConfigProvider>
```

## Density

Multiple densities can coexist on the page — each `ConfigProvider` writes its own wrapping `<div>` with `data-size`.

```tsx
<ConfigProvider size="compact">
  <DensePanel />
</ConfigProvider>
```

## Locale

```tsx
import { ConfigProvider, zhCN } from '@stcn52/cloud-ui'

<ConfigProvider locale={zhCN}>...</ConfigProvider>
```

## Custom tokens

Tokens live in `@theme` CSS-first. Override by declaring your own `@theme` *after* importing the library styles:

```css
@import "@stcn52/cloud-ui/styles.css";

@theme {
  --color-accent: oklch(0.68 0.18 160);
  --color-accent-weak: oklch(0.95 0.06 160);
  --color-accent-ink: oklch(0.36 0.14 160);
}
```
